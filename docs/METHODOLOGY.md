# Methodology & Justification

Agile methodology was selected for TransitLK because it supports iterative and incremental development, which is suitable for a system with multiple modules such as authentication, depot management, route planning, schedule management, fleet and driver management, fuel and maintenance logging, dashboard analytics, and reporting.

Since the project must be completed within a limited timeframe of one month, Agile allows the work to be divided into manageable sprints, ensuring steady progress and better time management. Each sprint delivers a working increment rather than deferring all software until the final week.

TransitLK is built as a MERN stack web application. Agile enables team members to work on different modules simultaneously, with backend APIs, frontend pages, and testing overlapping within the same sprint. This improves collaboration and efficiency across the team.

Additionally, Agile provides flexibility to accommodate changes during development through product backlog refinement. It also supports continuous testing — sprint review and testing in every iteration, plus integration, end-to-end, and user acceptance testing before release — helping to identify and fix issues early.

Therefore, Agile is an appropriate choice as it ensures organized development, reduces integration risks, and improves overall system quality.
